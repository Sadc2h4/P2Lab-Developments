# Page_specified.txt 説明書

## 概要

`Page_specified.txt` は、GitHub Pages ポータルのページ外観と GitHub 取得元をカスタマイズするための設定ファイルです。  
GitHub Pages リポジトリのルートに配置することで、ブラウザへの反映が自動で行われます。

---

## ファイルの配置場所

```
GitHub Pages リポジトリ/
├── Page_specified.txt         ← ここに配置（ルート直下）
├── index.html
├── style.css
├── github-loader.js
└── apps.js
```

---

## 記載ルール

### セクション（項目）の書き方

行頭に下記の記号を書くと「セクション（項目名）」として認識されます。  
記号の直後に半角スペースを入れても入れなくても動作します。

| 記号 | 書き方の例 |
|------|-----------|
| ◆ | `◆ ヘッダー色` または `◆ヘッダー色` |
| ■ | `■ ヘッダー色` |
| ● | `● ヘッダー色` |
| 【 】 | `【ヘッダー色】` |

### 値の書き方

セクション行の **次の行から** 値を記載します。  
空行はスキップされるため、セクション間に空行を入れても問題ありません。

```
◆ セクション名
値
```

### 値が複数行ある場合

グラデーション指定など、同じセクションに複数の値を記載する場合は改行して続けて書きます。

```
◆ ヘッダー色
#16171C
#0E335E
```

### 記載しなかった項目について

記載しなかったセクションはデフォルト値のまま変更されません。  
すべての項目を記載する必要はなく、変更したい項目だけ書けば大丈夫です。

## セクション一覧

### 【GitHub 取得元】カード表示に使う公開リポジトリの取得先

通常の GitHub Pages URL（`https://<account>.github.io/<repo>/`）では、`<account>` と `<repo>` を自動判定します。  
カスタムドメインを使う場合や、別アカウントの公開リポジトリを表示したい場合だけ指定してください。

| セクション名 | 別名として使える表記 | 説明 |
|------------|-------------------|------|
| `GitHubアカウント` | `github_owner`, `gh_owner` | カード表示に使う GitHub アカウントまたは Organization |
| `GitHub種別` | `github_owner_type`, `gh_owner_type` | `user` または `org` |
| `GitHub Pagesリポジトリ` | `ポータルリポジトリ`, `gh_repo` | この配布ページを置いているリポジトリ名。カード一覧から除外されます |
| `GitHubブランチ` | `github_branch`, `gh_branch` | `Page_specified.txt` や画像を GitHub API 経由で読む場合のブランチ |

**記載例：**
```
◆ GitHubアカウント
your-github-account

◆ GitHub種別
user

◆ GitHub Pagesリポジトリ
your-pages-repo
```

### 【ナビゲーションバー】上部の固定バー

| セクション名 | 別名として使える表記 | 説明 |
|------------|-------------------|------|
| `サイト名` | `ブランド名` | 左上に表示されるサイトタイトル文字 |
| `サイトアイコン` | `ブランドアイコン` | サイト名の左に表示される絵文字アイコン |
| `サイトアイコン画像` | `サイトアイコンファイル`, `favicon`, `site_icon_image` | サイト名の左に表示される画像とブラウザ用アイコン |
| `アクセントカラー` | `ボタン色` | ボタン・フィルタバー・リンクなどの強調色 |
| `ナビ色` | `ナビゲーション色` | 上部ナビバー自体の背景色 |

**記載例：**
```
◆ サイト名
（サイト名を記載）

◆ サイトアイコン
（絵文字を記載）

◆ サイトアイコン画像
favicon.ico

◆ アクセントカラー
#0071E3

◆ ナビ色
#16161A
```

---

### 【ヒーローエリア】タイトルと説明文が入る帯状のエリア

| セクション名 | 説明 | 補足 |
|------------|------|------|
| `ヘッダー色` | 背景色 | 1行 = 単色、2行 = グラデーション（上→下 左→右） |
| `ヘッダー画像` | 背景画像 | 指定した場合は `ヘッダー色` より優先される |
| `ヘッダー文字タイトル` | 大見出しのテキスト | |
| `ヘッダー文字色` | 大見出しの文字色 | |
| `ヘッダー文字説明` | サブ説明文のテキスト | |
| `ヘッダー文字説明色` | サブ説明文の文字色 | |
| `ヘッダー文字背景` | 文字背景の有無 | `あり` を指定すると、大見出しと説明文の背面に半透明背景を表示 |
| `ヘッダー文字背景色` | 文字背景の色 | 例: `#16161A` |
| `ヘッダー文字背景透過度` | 文字背景の透過度 | `0`〜`1`。例: `0.36` |

**記載例（色指定）：**
```
◆ ヘッダー色
#16171C
#0E335E

◆ ヘッダー文字タイトル
（大見出しテキストを記載）

◆ ヘッダー文字色
#FFFFFF

◆ ヘッダー文字説明
（サブ説明文を記載）

◆ ヘッダー文字説明色
#B0B3B8

◆ ヘッダー文字背景
あり

◆ ヘッダー文字背景色
#16161A

◆ ヘッダー文字背景透過度
0.36
```

**記載例（画像指定）：**
```
◆ ヘッダー画像
images/hero.png
```

---

### 【ページ背景】カード一覧が表示されるメインエリアの背景

| セクション名 | 別名として使える表記 | 説明 | 補足 |
|------------|-------------------|------|------|
| `背景色` | `background_color` | ページ全体の背景色 | |
| `背景画像` | `background_image` | ページ全体の背景画像 | 指定した場合は `背景色` より優先される。タイル状に繰り返し表示される |

**記載例：**
```
◆ 背景色
#F2F2F7

◆ 背景画像
images/bg-tile.png
```

## 画像の指定方法

### サイトアイコン画像 / favicon を使う場合

`サイトアイコン画像` に `.ico`、`.png`、`.svg` などの画像パスを指定できます。  
未指定で GitHub Pages リポジトリ内に `.ico` ファイルがある場合は、その `.ico` を自動的に画面上部のアイコンとブラウザ用 favicon に使用します。複数ある場合は `favicon.ico` を優先します。

```
◆ サイトアイコン画像
favicon.ico
```

### リポジトリ内の画像を使う場合

GitHub Pages リポジトリ内の画像ファイルへのパスを記載します。  
パスはリポジトリのルートからの相対パスで記載してください。

```
◆ ヘッダー画像
images/hero.png
```

上記の場合、GitHub Pages リポジトリ内の `images/hero.png` を参照します。

### 外部URLを使う場合

`https://` から始まるURLをそのまま記載します。

```
◆ ヘッダー画像
https://example.com/image.png
```

## 色の指定方法

CSSで使用できる色の書き方はすべて使用可能です。

| 形式 | 書き方の例 | 備考 |
|------|-----------|------|
| HEX（6桁） | `#0071E3` | 最も一般的 |
| HEX（3桁短縮） | `#07E` | |
| RGB | `rgb(0, 113, 227)` | |
| RGBA（透明度あり） | `rgba(0, 113, 227, 0.8)` | |
| CSS色名 | `navy` / `white` / `black` | |

---

## 推奨画像サイズ

| 用途 | 推奨サイズ | 縦横比 | 表示方法 |
|------|-----------|--------|---------|
| ヘッダー画像（`ヘッダー画像`） | **1920 × 220px** | 約 8.7 : 1 | 画面幅に合わせて拡縮（`cover`） |
| 背景タイル（`背景画像`） | **400 × 400px** | 1 : 1 | タイル状に繰り返し（`repeat`） |

> **ヘッダー画像について：**  
> 画面幅（1920px）に合わせて横いっぱいに広がり、縦は上下中央を基準にトリミングされます。  
> 重要なデザイン要素（文字・ロゴ等）は画像の中央付近に配置することを推奨します。

---

## 優先順位（色と画像が両方ある場合）

```
ヘッダー画像 を指定した場合 → ヘッダー色 は無視される
背景画像   を指定した場合 → 背景色   は無視される
```

---

## キャッシュについて

設定ファイルの変更はブラウザキャッシュにより、即時反映されない場合があります。

| 操作 | 方法 |
|------|------|
| 強制リロード（キャッシュ無視） | `Ctrl + Shift + R` |
| キャッシュを手動クリア | URLに `?gh_refresh=1` を付けてアクセス |
